# PacketPuncture Determinism Rules

PacketPuncture is built around strict deterministic behavior.

These rules define how artifacts must be generated to ensure reproducibility.

---

# Encoding Rules

All textual artifacts must use:

UTF-8 encoding with no BOM

Newline normalization:

LF only

No CRLF or CR line endings are permitted.

---

# JSON Serialization

JSON artifacts must be canonical.

Rules:

• stable key ordering  
• no unnecessary whitespace  
• deterministic serialization  
• consistent escaping behavior  

Any implementation must produce identical JSON output given identical inputs.

---

# Hashing

SHA-256 is used for all artifact hashing.

Hashes must be computed over the exact on-disk bytes of the artifact.

No normalization is allowed during hashing.

---

# Artifact Integrity

Each artifact generated by PacketPuncture must be verifiable through:

• SHA-256 hashing  
• cryptographic signatures  
• receipt log verification  

Artifacts must remain immutable once generated.

---

# Receipt Ledger Rules

Receipt logs are append-only.

Entries must contain:

• timestamp  
• action type  
• artifact hashes  
• previous log hash  
• sequence number  

This creates a verifiable ledger chain.

---

# Verification Behavior

Verification must be non-mutating.

Verification tools must:

• never modify artifacts  
• never repair artifacts silently  
• never rewrite evidence bundles  

Any repair process must be an explicit command producing a new artifact.

---

# Reproducibility

Given identical inputs and signing keys, PacketPuncture must produce identical artifacts.

Deterministic behavior ensures that independent verification across machines produces identical results.

---

# Determinism Enforcement

Tools and scripts within this repository must enforce these rules.

Any change that breaks deterministic behavior is considered a critical defect.
