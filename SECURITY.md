# Security Policy

PacketPuncture is designed as an evidence instrumentation tool for network events.

Security issues are taken seriously.

---

# Supported Versions

The current development branch is considered the supported version.

Security fixes will be applied to the latest maintained branch.

---

# Reporting Vulnerabilities

If you discover a security vulnerability, please report it privately.

Include the following information:

• description of the vulnerability  
• steps to reproduce  
• affected components  
• potential impact  

Please do not publicly disclose vulnerabilities until they have been reviewed.

---

# Security Design Goals

PacketPuncture prioritizes:

• deterministic artifact generation  
• verifiable evidence chains  
• append-only receipt logs  
• cryptographic authenticity  

The system is not designed to perform packet inspection security analysis itself.

Its purpose is to produce verifiable evidence artifacts.

---

# Responsible Disclosure

Researchers who responsibly disclose vulnerabilities will be credited unless anonymity is requested.
